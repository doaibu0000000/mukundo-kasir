# Rencana Konversi ke GitHub Pages (100% Client-Side, IndexedDB)

## Strategi
Membuat **lapisan backend baru sepenuhnya dalam JavaScript** yang berjalan di browser, menggantikan Python/Flask. Frontend (UI, CSS, view JS) **tidak diubah** — hanya `api.js` (lapisan transport) yang diganti implementasinya dari `fetch()` ke pemanggilan IndexedDB langsung.

## Yang TIDAK berubah
- `index.html`, `neu.css`, semua `js/views/*.js`, `theme.js`, `helpers.js`, `notify.js`, `state.js`, `router.js`, `app.js` — **UI 100% sama**
- Object `window.API` dan signature method-nya **identik** → semua view tetap bekerja

## File baru yang dibuat (semua di folder `frontend/`)

### 1. `js/db.js` — IndexedDB layer
- Membuka database `pos-db` dengan 6 object stores (mirip tabel SQLite): `users`, `categories`, `products`, `transactions`, `settings`, `txCounters`
- Seeding data awal (admin/kasir, kategori, produk sampel) saat DB baru
- Auto-increment ID manual (counter per store)
- Helper CRUD generik: `getAll`, `get`, `put`, `delete`, `query`

### 2. `js/backend.js` — replikasi semua logika Python service/repository
Method-by-method port dari Python ke JS:
- **Auth**: login/logout, session di localStorage (`pos.user`), RBAC check
- **Products**: list/search/get/create/update/delete/toggle + validasi (nama wajib, minimal 1 varian harga)
- **Categories**: list/create/update/delete + validasi nama duplikat
- **Transactions**: 
  - `generateOrderId()`: format `<PREFIX>-<YYMMDD>-<NNNN>` (counter per hari)
  - `calcDiscount()`: percent [0-100] / amount [0-subtotal]
  - `checkout()`: validasi cart, hitung subtotal/discount/grand_total/profit, simpan header+details (atomic)
  - `confirmCheckout()`: kurangi stok per varian
  - `cancelCheckout()`: hapus transaksi pending
- **Reports**: `dashboardSummary()` (today + month turnover/profit/count, low stock ≤5), `dailyRevenue(days)` (isi hari kosong dgn 0), `financialReport()`
- **Settings**: get/update, QR/logo upload (simpan base64 di IndexedDB)
- **Image handling**: gambar produk disimpan sebagai base64 data URL di IndexedDB (bukan file server)

### 3. `js/api.js` — di-rewrite (signature tetap sama!)
- Object `window.API` dengan **method & parameter identik** (listProducts, checkout, dll)
- Alih-alih `fetch()`, setiap method memanggil `Backend.*` langsung
- `async` interface tetap → semua view tetap bekerja tanpa perubahan
- Error throwing tetap sama (throw Error untuk ValueError/PermissionError)

## File yang diubah

### 4. `frontend/index.html`
- Hapus `<script src="https://cdn.tailwindcss.com">` → ganti Tailwind **play CDN** (versi yang berfungsi di file:// maupun GitHub Pages) — sebenarnya CDN saat ini sudah jalan di Pages, jadi mungkin tidak perlu diubah
- Tambah `<script defer src="js/db.js">` dan `<script defer src="js/backend.js">` SEBELUM `api.js`
- Hapus/tahan service worker untuk mode IndexedDB (SW caching tidak perlu karena semua data lokal)
- Pertahankan meta tag PWA (manifest, icons, theme-color)

### 5. `js/views/transaction.js` — bagian cetak struk
- `printReceipt()`: ganti dari `API.printReceipt(data)` (Win32) → render struk ke `#print-area` lalu panggil `window.print()`
- Gunakan CSS `@media print` yang **sudah ada** di neu.css (58mm, white background)
- Dialog cetak browser muncul → user pilih printer → cetak (tetap jalan di komputer dengan printer thermal)
- `buildReceiptHTML()` yang sudah ada langsung dipakai untuk isi `#print-area`

### 6. `js/views/settings.js` — bagian printer
- Sembunyikan/ubah seksi "Pengaturan Printer" (pilih printer OS, tes koneksi, cetak tes) karena tidak relevan di mode browser
- Ganti jadi catatan: "Pencetakan struk menggunakan printer default browser"
- Pertahankan pengaturan lebar kertas (58/80mm) — ini dipakai untuk `@page size` di CSS print

### 7. `js/app.js` — minor adjustment
- `boot()`: ganti `await API.ping()` → inisialisasi DB (`await Db.init()`) dulu
- Hapus dependency server (API.ready check)

### 8. `manifest.json`
- `start_url` tetap `/` (root GitHub Pages akan jadi `username.github.io/kasir/`)
- `scope` tetap `/`

## File Python yang tidak dipakai (tetap di repo untuk referensi, atau bisa diabaikan)
- `app.py`, `server.py`, `api/bridge.py`, `config/`, `models/`, `repositories/`, `services/` — tidak dijalankan di GitHub Pages
- Bisa dipindah ke folder `server-reference/` agar tidak membingungkan

## Konsekuensi (sudah disepakati user)
- ✅ **1 browser = 1 database** (data tidak sync antar device)
- ✅ **Cetak struk**: muncul dialog `window.print()` dulu, pilih printer, lalu cetak (bukan silent)
- ✅ **Multi-user**: admin & kasir pakai browser yang sama (switch login)
- ✅ **Hosting**: GitHub Pages, gratis, 24 jam online
- ✅ **URL**: `https://username.github.io/kasir`

## Urutan implementasi
1. Buat `js/db.js` (IndexedDB + seeding)
2. Buat `js/backend.js` (port semua logika)
3. Rewrite `js/api.js` (panggil backend, signature sama)
4. Ubah `js/views/transaction.js` (cetak via window.print)
5. Ubah `js/views/settings.js` (bagian printer)
6. Ubah `js/app.js` (boot tanpa server)
7. Update `index.html` (script tags, pertahankan PWA)
8. Buat `index.html` root untuk GitHub Pages redirect (jika perlu)
9. Uji coba lokal (buka index.html langsung / jalankan statis)
10. Update `.gitignore` + `README.md` (instruksi deploy ke GitHub Pages)
